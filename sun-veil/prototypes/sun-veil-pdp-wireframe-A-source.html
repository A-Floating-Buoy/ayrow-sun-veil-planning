모바일 우선 와이어. 데스크탑은 같은 컨텐츠를 2~3열 그리드로 확장하는 것을 전제로 합니다.

역할

카테고리 재정의 선언. "필름이 아닌 멤브레인"이라는 단일 대비축을 첫 화면에 박는다. 사용자가 카페24 상단 모듈에서 가격·옵션을 본 직후 첫 인상을 만드는 자리.

UI Elements

  • Full-bleed hero visual (모바일 3:4, 데스크탑 4:5)
  • H1 + sub-headline (원문 그대로)
  • Kicker chip — Aesthetic Guard

인터랙션 / 반응형

진입 시 "Film" → "Membrane" 짧은 텍스트 전환. prefers-reduced-motion에서는 정적 표시. 데스크탑은 좌 비주얼 + 우 텍스트 2열로 확장.

비주얼 애셋

  • 모델 착용 컷 — 광대존 close-up, 자연광
  • 대안 1: 손 위 패치 매크로 (투명도 강조)
  • 대안 2: 제품 + 그림자만으로 구성한 정물

검수 / 약점

카페24 상품 모듈이 이미 제품명·가격을 보여주므로 제품명 반복은 불필요. 첫 viewport에 신뢰 chip(0.004mm, UPF 50+, Dermatest)을 동반 노출할지 결정 필요.

Category Reframing — Film vs Membrane

사용자 질문 — 기존 선패치와 뭐가 다른가?

같은 ‘패치’로 보였다면, 완전히 잘못 보고 있던 겁니다. 막는 것 vs 숨 쉬는 것
기존 선패치 필름 = 막 공기 흐름 차단 / 열·습기 정체
에이로우 선베일 나노 섬유 멤브레인 공기·수분 흐름 유지 / 장시간 부착 가능
데이터 블록 · 투습도 12,356 / 공기투과 <1.0 / 기공 0.7μm
데이터 블록 · UVB 99.9 / UVA 95.3 / UPF 50+

역할

Hero의 선언을 증명으로 잇는 자리. "왜 다른가"를 비교표와 데이터로 박는다. 카테고리 재정의가 머리에 남는 결정적 섹션.

UI Elements

  • Comparison cards 2열 (모바일 세로 스택)
  • 데이터 블록 카드 x2 (Breathable / UV)
  • 섹션 인트로 카피 + 비교 → 데이터 순서

인터랙션 / 반응형

스크롤 진입 시 비교 카드 한 줄씩 활성화. 데이터 블록의 수치는 진입 시 1회 count-up. 모바일은 비교 카드가 세로 스택되어 전체 길이 +30% 추정.

비주얼 애셋

  • 필름 vs 멤브레인 마이크로 일러스트 (단면도)
  • 대안: 현미경 이미지 (실측 자료 필요)
  • 데이터 블록은 텍스트 카드로 충분, FITI 로고는 보조 마크로 작게

검수 / 약점

원문은 5개 항목 비교 — 모바일에서 세로로 펼쳐지면 길어 인지 피로. 데이터 블록(투습/UV)이 섹션 08(Proof)와 중복. UV-A 수치는 원문 96.4% / 리포트 95.3% 불일치 — 95.3%로 통일 필요.

역할

스크롤을 내리기 전에 핵심 의심을 빠르게 제거. Yes-Yes-Yes…-No 리듬으로 신뢰를 누적시키고 마지막 NO("티 나나요?")가 반전 펀치라인으로 작동하는 의도된 카피 장치.

UI Elements

  • Accordion 6개 (탭하면 답변 펼침)
  • 각 항목 좌측에 큰 YES/NO 라벨 (시각적 리듬)
  • 일부 항목 답변 내부에 → "사용 장면 보기" 인라인 트리거

인터랙션 / 반응형

한 번에 한 항목 펼침 (단일 활성). YES/NO 라벨은 펼침 여부와 무관하게 항상 노출되어야 Yes-set 리듬 유지. 데스크탑은 2열 그리드도 가능하지만 리듬 유지를 위해 1열 권장.

비주얼 애셋

  • "메이크업 위 부착" 짧은 시연 영상 (10초 silent loop)
  • "투명 구조" close-up 매크로 컷

검수 / 약점

YES 5개는 다소 길어 리듬 약화 가능 — 4 YES + 1 NO가 더 임팩트. 단, 의도된 장치이므로 변경 시 카피 라이터와 합의 필요. Full FAQ(09)와 4~5개 질문 중복 — 의도된 페어링이지만 UI에서 핸드오프 명시 필요.

Problem / Pain Point

사용자 질문 — 정확히 어디가 문제인가?

이 자리, 바르는 선케어만으로 충분할까요? 선케어가 가장 먼저 무너지는 자리 — 눈가·광대
얼굴 일러스트 · 광대존 hotspot 강조

타이밍이동·약속 사이

예민·점착2차 자극 우려

역할

"왜 하필 광대·눈가인가" — 부위 특이성으로 제품의 필요성을 좁힘. 광대존이 자외선·메이크업·표정 움직임 모두에서 가장 먼저 무너지는 지점임을 환기.

UI Elements

  • 얼굴 일러스트 + 광대존 hotspot
  • Pain moment card grid (모바일 2열, 데스크탑 3열)
  • 섹션 인트로 카피("이 자리, 충분할까요?") + 시각 + 카드

인터랙션 / 반응형

얼굴 일러스트의 hotspot을 탭하면 해당 pain point 카드가 강조. 모바일에서는 hover 없으므로 자동 순환 또는 정적 표시. 데스크탑은 hotspot ↔ 카드 양방향 호버.

비주얼 애셋

  • 얼굴 일러스트 1장 — 광대존 강조 (브랜드 톤의 라인 드로잉)
  • 모먼트별 라인 픽토그램 6~7종
  • 대안: 일러스트 대신 실제 모델 광대존 close-up 사진

검수 / 약점

원문 7개 모먼트(메이크업·타이밍·땀·감성·컨디션·예민·점착) — 섹션 03 모먼트(9개)와 의미 중복 큼. 두 섹션이 합치면 거의 동일한 메시지. 통합 필요.

How It Works

사용자 질문 — 어떻게 보호하고 어떻게 붙나?

두 개의 레이어가 하나의 피부처럼 작동합니다 How It Protects + How It Bonds
2-Layer Exploded Diagram · Layer1 멤브레인 + Layer2 약액층
Layer 1초박막 투명 멤브레인 (자외선 보호 층)
Layer 2약액층 (외부 자극 환경에서 컨디션 유지 서포트)
Moisture-Bonding 다이어그램 · 점착제 없는 클린 밀착

역할

"어떻게 작동하는가"의 구조적 답. 두 레이어 + Moisture-Bonding 방식으로 차단·밀착의 원리를 시각화. 카테고리 차별성을 구조 사실로 뒷받침.

UI Elements

  • Layer split exploded view (정적 일러스트)
  • Layer 카드 2개 (멤브레인 / 약액층)
  • Bonding 다이어그램 (단면 컷)

인터랙션 / 반응형

스크롤 진입 시 두 레이어가 8~12px 분리되어 구조 강조. 클릭 시 각 레이어 설명 펼침. 추후 3D 애셋 준비되면 교체 가능, 그 전에는 정적 SVG로 유지.

비주얼 애셋

  • 2-layer exploded illustration (정적 SVG)
  • Moisture-Bonding 단면 일러스트 또는 매크로 사진

검수 / 약점
